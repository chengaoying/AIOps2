## Vision

### 10x Check
从\"诊断工具\"升级为\"智能运维平台\"：
- 告警触发诊断：作业失败 → 自动诊断 → 推送通知（无需手动）
- 统一仪表板：一站式查看4个组件的健康度和诊断结果
- 自愈能力：AI诊断 → 一键修复 → 自动验证

### Platonic Ideal
想象运维工程师的日常：
- 早上9点：打开Dashboard，看到集群健康度评分90/100
- 10点：收到钉钉告警\"Spark Executor OOM\"，点击通知直接看到诊断结果
- 11点：点击\"一键修复\"，Executor内存从4G增加到6G，作业自动重试成功
- 下午：查看本周MTTR下降20%的趋势报告

## Accepted Scope (added to this plan)

### Phase 1 Extended (新增功能)

#### 1. 告警触发诊断
- 告警规则引擎（监控作业失败事件）
- 自动触发诊断流程
- 通知渠道集成（钉钉/飞书/企业微信/邮件）
- 通知模板（诊断结果摘要+一键操作链接）

#### 2. 统一运维仪表板
- 集群健康度评分（基于作业成功率）
- Top失败作业列表（按组件分组）
- 实时告警面板
- 诊断历史追溯
- 趋势图表（失败率、响应时间）

#### 3. 自愈/自动化修复
- 修复脚本模板（预定义操作）
- 审批流程（高风险操作需确认）
- 修复后验证（自动检查作业是否恢复）
- 回滚机制（如果修复失败）

### Phase 1 Core (原有功能)

- 数据采集层（YARN/Hive/Spark/Flink插件）
- 元数据仓库（StarRocks）
- AI诊断引擎（规则+LLM混合）
- 用户交互层（React UI）

## Technical Considerations from Review

### Architecture
- 单点故障：StarRocks HA、Kafka副本、诊断引擎降级模式
- 数据流边界：本地缓冲、DLQ、LLM降级策略

### Security
- API认证：JWT + refresh token
- Webhook签名验证（HMAC）
- LLM数据脱敏

### Performance
- 异步诊断模式（立即返回incident_id）
- 结果缓存（相似症状）
- 并发控制（令牌桶限流）
