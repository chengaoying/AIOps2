## 1. 架构评审

### 1.1 数据流架构调整

**问题**: 设计文档中使用Kafka作为消息队列，但分区设计不足（12分区），且增加了系统复杂度。

**决策**: ✅ 去掉Kafka，Collector Agent直接写入StarRocks
- Collector Agent实现本地WAL缓冲（内存队列10000条 + 磁盘WAL）
- StarRocks不可用时写入本地WAL，重启后恢复
- 背压控制：WAL超过1GB时丢弃最旧数据

**影响**: 
- 简化架构，减少一层中间件
- 降低运维复杂度
- 需要在Agent中实现容错机制

---

### 1.2 LLM安全防护

**问题**: 设计文档提到"LLM数据脱敏"，但缺少Prompt Injection防护设计。

**决策**: ✅ 实现三层防护
1. **输入验证**: 长度限制（10000字符）、危险字符过滤、HTML转义
2. **Prompt结构化**: 系统指令与用户数据用明确分隔符隔离（```SYSTEM```/```USER_DATA```）
3. **输出验证**: 检查LLM返回是否包含危险命令

**实现示例**:
```go
func SanitizeInput(raw string) string {
  if len(raw) > 10000 { raw = raw[:10000] }
  dangerous := []string{"```", "<script>", "${"}
  for _, d := range dangerous {
    raw = strings.ReplaceAll(raw, d, "")
  }
  return html.EscapeString(raw)
}
```

### 1.4 规则引擎必要性

**问题**: 设计文档提到"规则+LLM混合诊断"，但未说明规则引擎的价值。

**决策**: ✅ 规则+LLM混合，降低70%成本
- 常见故障（Spark OOM、Hive语义错误）占70%+，用规则匹配免费且快速（<1秒）
- LLM调用成本高（每次约0.1-0.5元），只用于复杂场景
- 规则引擎使用前缀树索引，匹配时间<10ms

**成本对比**:
- 纯LLM方案：每天1000次诊断 = 100-500元/天 = 3000-15000元/月
- 混合方案：70%规则（免费）+ 30%LLM = 30-150元/天 = 900-4500元/月
- **节省成本：70%**

// LLM调用失败降级到规则引擎
func (d *DiagnosisService) Diagnose(symptom string) (*Result, error) {
  // 先尝试规则引擎
  if result, err := d.ruleEngine.Match(symptom); err == nil {
    return result, nil
  }
  
  // 规则未命中，调用LLM
  result, err := d.llm.Diagnose(symptom)
  if err != nil {
    log.Warn("LLM调用失败，降级到规则引擎", "error", err)
    metrics.LLMDegradationTotal.Inc()
    return d.ruleEngine.FallbackDiagnose(symptom)
  }
  
  return result, nil
}
```

type BatchWriter struct {
  queue    chan Event
  batch    []Event
  ticker   *time.Ticker
  storage  Storage
}

func (w *BatchWriter) Start() {
  w.ticker = time.NewTicker(5 * time.Second)
  
  for {
    select {
    case event := <-w.queue:
      w.batch = append(w.batch, event)
      if len(w.batch) >= 1000 {
        w.flush()
      }
    case <-w.ticker.C:
      if len(w.batch) > 0 {
        w.flush()
      }
    }
  }
}

func (w *BatchWriter) flush() {
  compressed := gzip.Compress(w.batch)
  w.storage.StreamLoad(compressed)
  w.batch = w.batch[:0]
}
```

func (c *CacheService) GetOrDiagnose(symptom string) (*Result, error) {
  // 计算症状指纹
  fingerprint := hash(symptom)
  
  // 查询缓存
  if cached, err := c.redis.Get(fingerprint).Result(); err == nil {

}

#### 4.1.4 LLM并发控制
```go
type RateLimiter struct {
  bucket *rate.Limiter
}

func NewRateLimiter() *RateLimiter {
  return &RateLimiter{
    bucket: rate.NewLimiter(rate.Limit(1), 10), // 每秒1个，容量10
  }
}

func (r *RateLimiter) Diagnose(ctx context.Context, symptom string) (*Result, error) {
  // 等待令牌
  if err := r.bucket.Wait(ctx); err != nil {
    return nil, ErrRateLimitExceeded
  }
  
  // 调用LLM，30秒超时
  ctx, cancel := context.WithTimeout(ctx, 30*time.Second)
  defer cancel()
  
  result, err := r.llm.Call(ctx, symptom)
  if err == context.DeadlineExceeded {
    // 超时降级到规则引擎
    return r.ruleEngine.Diagnose(symptom)
  }
  
  return result, err
}
