设计原则

功能优先

简单易用，减少运维压力，所有元素服务功能

实用主义

视觉噪音归零，无渐变、无阴影、无装饰性图形

快速高效

信息密度高但层次清晰，一眼识别关键信息

用户痛点

痛点现状期望
排查耗时长 登录多系统，手动查日志 一键诊断
跨平台复杂 YARN/Hive/Spark/Flink 各不同 统一视图
根因不明确 日志分散，难以定位 AI 自动分析
修复方案模糊 需要自行研究 给出具体建议

核心功能

F1: Dashboard 首页

今日概览统计 · 平台分布可视化 · 最近诊断列表

F2: 作业诊断 P0

输入作业ID/日志 → 知识库检索 → LLM 推理 → 根因+建议

F3: 诊断历史

按时间/平台/状态筛选 · 查看历史详情

F4: AI 助手

自然语言查询 · 多轮对话

F5: 系统配置

用户管理 · 集群配置 · 系统配置

核心架构决策

决策选择理由
消息队列 去掉 Kafka 简化架构，降低复杂度
诊断策略 知识库+LLM 可维护性强，覆盖面广
LLM 安全 三层防护 输入验证+结构化+输出验证
高可用 无状态微服务 便于扩展和故障恢复
知识库 Vector+全文 高效相似度匹配
自愈系统 Phase 2 Phase 1 聚焦诊断核心
